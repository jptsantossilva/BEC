# BEC Agent Guide

## Project Overview
BEC is a Python 3.12 automated Binance spot trading app with Streamlit dashboard,
scheduled jobs, backtesting, market-phase ranking, Telegram notifications, AI
analysis, Docker deployment, and VitePress docs.

Keep this file brief and focused. Put longer task-specific guidance in
`agent-docs/` and link to it from here.

## Repository Layout
- `bec/` contains the main application package.
- `pages/` contains Streamlit pages loaded by the dashboard.
- Root Python files such as `dashboard.py`, `jobs_runner.py`, and `main.py` are
  compatibility wrappers for Docker, Streamlit, cron, and scheduler commands.
- `docs/` contains the VitePress documentation site.
- `docker/`, `Dockerfile`, and `docker-compose.yml` define container runtime.
- Runtime files such as `data.db`, `main.log`, and generated backtest output are
  local state, not source code.

## Setup and Commands
- A local Python virtual environment exists at `.venv/`; prefer
  `.venv/bin/python` and `.venv/bin/streamlit` when running project commands.
- Install Python dependencies: `python -m pip install -r requirements.txt`
- Run dashboard locally: `streamlit run dashboard.py --server.port=8080`
- Run scheduler locally: `python jobs_runner.py`
- Run docs locally: `npm run docs:dev`
- Build docs: `npm run docs:build`
- Start Docker stack: `docker compose up -d`
- Inspect Docker logs: `docker compose logs -f dashboard` or
  `docker compose logs -f jobs_runner`

`requirements.txt` is generated by `pip-compile`; do not hand-edit dependency
pins unless the dependency workflow is being updated deliberately.

## Testing and Validation
- Run unit tests: `python -m pytest tests`
- Compile core Python files:
  `python -m py_compile bec/*.py bec/utils/*.py bec/exchanges/*.py bec/signals/*.py`
- For docs changes, run `npm run docs:build`.
- For Streamlit UI changes, run the dashboard and inspect the affected page.
- For Docker/runtime changes, verify the relevant service starts and logs cleanly.

## Trading and Database Safety Rules
- Never commit `.env`, Binance keys, Telegram tokens, OpenAI keys, `data.db`, or
  other local runtime secrets/state.
- Do not run `docker compose down -v` unless the user explicitly asks to delete
  persistent data.
- Treat order execution, balance handling, position sizing, stop loss, take
  profit, and symbol selection as high-risk code. Add focused tests or a clear
  manual validation note when changing these areas.
- Preserve root wrappers unless intentionally changing Docker, cron, Streamlit,
  or scheduler compatibility.
- See `agent-docs/trading-safety.md` before changing trading or persistence
  behavior.

## Documentation Workflow
- User-facing docs live in `docs/` and should build with `npm run docs:build`.
- Keep README installation and Docker commands aligned with `docker-compose.yml`.
- Add screenshots or generated reports only when they are intentionally part of
  the public docs.
- Unless a task explicitly specifies another language, all user-facing app text,
  captions, labels, docs, and messages must be written in English only.

## Known Gotchas
- Runtime state may exist in the repo working tree during local development;
  avoid modifying or deleting it unless the task requires it.
- Root scripts are usually thin wrappers around `bec.*` modules. Prefer changing
  package code under `bec/` and keeping wrappers stable.
- Backtesting and Monte Carlo logic can be deterministic only when seeds or
  fixtures are controlled.
